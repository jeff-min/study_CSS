Flexbox

        상대적으로 어려운 float 이나 position 을 미리 해본것 .
        웬만하면 Flexbox 만 사용해도 된다.

        Flexbox 는 일단
        float 이나 position 할때는 프로세스에대해 어떻게 동작하는지 하나하나 설명 했지만.

        Flexbox는 너무 완벽하다.  너무 잘 만들어져서

        

        How to use it     // 잘 사용하는 법이 중요하다

            Flexbox 사용해서 정렬할때는 4가지 사고 방식만 기억하면 된다. (4 Step)

            1. Flexbox 를 쓸거라는 선언!
            2. 가로로 정렬인지 ?세로 정렬인지? 정하는 것
            3. 한줄안에 다 정렬 할 지 / 상황에 따라 여러줄을 만들어 줘야 할 지

            4. Flexbox 사용과정

            1~~3 . Flexbox 준비과정

            
            .flexbox {

            1.  display : flex

            2.  display : inline-flex
            }
            

            flex            - > block 이랑 비슷하지만 // block 은 할 수 없는 요소를 쉽게 정리 할 수 있는 특징이 있다.
            inline-flex     - > inline-block 이랑 비슷하지만 // inline-block 은 할 수 없는 특징이 있다.

            
            정렬을 하고자 하는 요소를 감싸고 있는 부모 박스에게 선언을 하자!


            
            .flexbox {

                flex-direction : row;

                 
                row | row-reverse | column | column-reverse


                row                     - 가로 방향 정렬 
                
                column                  - 세로 방향 정렬
                
                -reverse 는 반대 방향 
                
                


                flex 를 사용하면 보이지 않는 2개의 축 ( Axis ) 이 생긴다
                이 Axis 가 flex-direction이 어떤 (값)방향으로 설정 되었는지에 따라 2개의 축 방향이  완전히 달라진다.
                }
                

                일단 )
                Main aixs  와  Cross axis 가 생기는데

            먼저 Main aixs 는 flex-direction 의 방향에 따라 생기고
                Cross axis 는 Main aixs 와 정확히 수직을 이룬다.



            
            .flexbox {

            flex-wrap : nowrap;

            
            nowrap | wrap 
                }
                

                1. nowrap            - >   여러줄 생성 X / 감싸지wrap 않고 자식의 사이즈를 줄여서라도 한 줄로 정렬해버리는 flex-wrap : nowrap
                
                2. wrap              - >   자식 요소가 절대 줄어들지 않아야 할 때 / 한 줄에 모두 정렬하기에 공간이 넉넉치 않으면 여러 줄을 만들어 자식 요소 사이즈를 유지하는 flex-wrap : wrap




                

        

                justify-content : 속성값        center | flex-start | flexr-end | space-between | space-around

                center                      :   정렬 하고 싶은 방향으로 가운데 정렬
                flex-start                  :   시작점으로 정렬!! // EX) row 왼쪽에서 오른쪽으로 가는방향이면  시작점은 왼쪽이니 [ 왼쪽정렬 ]
                flexr-end                   :   도착점으로 정렬 !!  // EX ) 같은 상황이면 [ 오른쪽 정렬 ]
                space-between               :   요소들 사이에 간격을 부모 width 에서 자식 width을 빼고 나머지 남은 간격을 균등하게 요소 사이사이 공간을 만들어줌.
                space-around                :   요소들 사이에 간격을 같게 해줌 // EX ) margin 간격을 동일하게 준거 처럼.                            

                align-items     : 속성값          같음.

                
                align-items | align-content 의 차이는 ?

                
                
                

                


                align-content  은   justify-content 와 같게 전체를 하나만의 (흐름) cross axis 축으로  같게 하고 싶다 .  ///[방향과 정렬을 하는 flex line을 생성.]
                

                align-items    은   줄이 생긴만큼 flex line 을 만들어내고   // 즉 방향과 정렬성은 같으나 (흐름이)cross axis 여러개 생성
